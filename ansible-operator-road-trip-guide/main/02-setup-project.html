Setup the project

Log in to the registry you use

You need the an account to use a container registry as the repository for your operator manifests bundle. Then use those credentials to log in using docker/podman.

For instance you can create a free account in quay.io 
docker login quay.io
Before continuing with the guide please run this command with the proper value (your user in the registry). 
export REG_USERNAME=%REG_USERNAME%
Username: <registry_username>
Password:
Login Succeeded

Define environment variables

Letâ€™s define some environment variables that will come handy later. For the sake of simplicity we’ll create a script to load environment variables.

All lab should happen inside a folder…​ please think of one and subtitute in the next snippet, then run it. 
export LAB_FOLDER=%LAB_FOLDER%
mkdir ${LAB_FOLDER} && cd ${LAB_FOLDER}
You can change the APP_NAME, ORGANIZATION. BUT definitely you want to change USERNAME with the container registry account user name you have. 
cat << 'EOF' > ./settings.sh

export APP_NAME=minerva
export OPERATOR_NAME=${APP_NAME}-operator
export OPERATOR_IMAGE=${OPERATOR_NAME}-image

export ORGANIZATION=atarazana
export DOMAIN=${ORGANIZATION}.com
export API_VERSION=${APP_NAME}.$atarazana.com/v1

export PROJECT_NAME=${OPERATOR_NAME}-system

export USERNAME=%REG_USERNAME%

export VERSION=0.0.1
EOF

Load environment:

chmod u+x ./settings.sh
. ./settings.sh
